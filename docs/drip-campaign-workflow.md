# Drip Campaign Workflow

Use this workflow when Codex is planning a Brevo welcome sequence, lead nurture, quote follow-up, winback, review request, or similar lifecycle campaign.

Codex prepares the campaign. The user builds, tests, and activates it manually inside Brevo.

## Intake

Gather or infer:

- Project or business name.
- Audience.
- Entry trigger.
- Conversion goal.
- Offer.
- Voice and compliance constraints.
- Number of messages.
- Timing.
- Required contact attributes.
- Required lists, segments, forms, templates, and backend events.

## Output Sections

Use this structure:

1. Campaign Brief.
2. Audience And Trigger.
3. Sequence Plan.
4. Message Drafts.
5. Brevo Assets Needed.
6. Backend Requirements.
7. QA Checklist.
8. Manual Brevo Handoff.

## Sequence Plan Fields

For each message, include:

- Delay or timing.
- Subject.
- Preheader.
- Goal.
- CTA.
- Personalization fields.
- Success metric.

## Backend Requirements

When the drip depends on a form or event, specify:

- Endpoint or event source.
- Contact attributes.
- Brevo list IDs or list names to confirm.
- Spam controls.
- Duplicate contact behavior.
- Server-side secret requirements.

## Manual Brevo Handoff

End with concrete Brevo dashboard steps:

- Create or confirm the list or segment.
- Create templates from the copy.
- Configure automation trigger and delays.
- Add conditions or exclusions.
- Send tests from Brevo.
- Review tracking, unsubscribe, sender, and domain settings.
- Manually activate the automation in Brevo.
